# Ninja Slayer 牌池设计审计

> 审计日期：2026-07-13  
> 方法：采用 [卡组构筑游戏卡牌设计指南](card-design-guide.md) 的术语、设计流程与四级检查标准。  
> 数据基线：[忍者杀手卡牌分类参考](card-catalog.md)，并以 `Cards/*.cs`、相关 `Powers/*.cs`、`Content/*.cs` 和 `localization/zhs/cards.json` 交叉核对。  
> 边界：本文评价当前结构并提出原则性方向，不提出具体改牌文本或新数值。

## 1. 审计口径

### 1.1 覆盖范围

项目中存在 87 个非模板卡牌源码文件，与卡牌目录总数一致：

| 获取类型 | 数量 | 是否按普通奖励牌评估 |
|---|---:|---|
| 初始 | 4 | 否；按起始套装与成长压力评估 |
| 白（Common） | 15 | 是 |
| 蓝（Uncommon） | 35 | 是 |
| 金（Rare） | 25 | 是 |
| 先古（Ancient） | 3 | 否；按特殊获取评估 |
| Token | 2 | 否；成本计入生成它们的卡 |
| Status | 3 | 否；按系统资源或负担评估 |
| **合计** | **87** | **普通奖励候选共 75 张** |

“普通奖励候选”按稀有度与注册语义划分；实际奖励投放仍由游戏的卡池与奖励规则决定。矩阵中的“裸用”是设计层面的相对判断，不是胜率结论：

- **高**：不依赖专属组件也能完成主要职责。
- **中**：有基础用途，但价值明显取决于时点、目标或常见配合。
- **低**：主要价值需要专属输入、特定状态或未来组件。
- **不适用**：生成牌、状态牌或特殊系统对象，不参与普通裸抓。

### 1.2 缩写

| 缩写 | 含义 |
|---|---|
| 攻 / 防 / 牌 / 能 | 进攻、防御、过牌、能量四个能力维度 |
| 入 / 出 | 核心机制的输入 / 输出 |
| 回路 | 同时读取并加强核心机制，可自我放大 |
| 导出 | 把一个体系的积累转化到另一能力维度 |
| 桥 | 连接两个体系或底层机制 |
| 过渡 | 主要依靠裸值、针对性或通用功能工作 |

## 2. 执行摘要

### 2.1 当前牌池已经具备的优势

1. **三个主机制都能真实改变基础战斗。**手里剑改变出牌次数、抽牌和消耗堆；茶道改变手牌占用、回合能量与花费节奏；空手道是会被攻击逐层兑现的可消耗标记。这符合指南 §4.3“机制应先有实际意义，再谈监听触发”的要求。
2. **自然协同数量可观。**手里剑天然连接攻击次数、多段、抽牌、消耗和手牌；茶道天然连接保留、能量、手牌容量、弃牌和消耗；空手道天然连接多段攻击、标记积累、结算与敌我双方风险。
3. **跨体系桥梁已经形成网络。**`喝茶剑`把手里剑导向茶道，`注入`把手里剑导向空手道，`殴人茶`把茶道导向空手道；`半月圆规踢`、`我打`、`进化`等又从茶道或状态行为导出攻、防、能量。
4. **四个能力维度都有覆盖。**进攻最充足；防御在三个主体系中都有入口；能量以茶道为中心并有若干系外工具；过牌可由手里剑自循环、茶道牌和通用过滤共同提供。

### 2.2 主要结构风险

1. **共同主题已经出现，但尚未被清晰收束。**最接近当前牌池事实的玩法主题是：  
   **“把暂存于手牌、敌我标记和形态中的资源，通过高频攻击与正确的释放时点转化为攻、防和运转。”**  
   手里剑、茶道和空手道都能解释这句话；奈落则强调高频攻击的爆发与燃烧代价。但“其他”区仍有较多独立能力，部分牌只与负面效果、回避、追击或附魔局部互动，削弱了奖励选项之间的整体关联。参见指南 §6.3。
2. **启动负担可能高于目录表面显示。**75 张普通奖励牌中有大量能力牌和高稀有度桥梁；多个体系要先支付 1～3 费能力或拿到金卡，才从“能工作”变成“能互相工作”。通用低成本过滤不少，但无条件的大幅过牌较少，可能出现上限充足、首轮笨重的结构。
3. **关键跨体系连接偏向金卡或能力牌。**`喝茶剑`、`注入`、`殴人茶`、`茶道三昧`、`剑来`、`空手道滚石`等都位于金卡。这能制造高上限，却降低普通奖励中多个选项同时相关的频率。体系结合若经常等待稀有桥梁，会把决策的一部分交给发牌运气。
4. **部分输出是纯战未来或狭窄监听。**例如按生成茶道、弃置茶道、消耗茶道或手里剑未格挡伤害触发的能力，在没有对应输入时价值很低。它们并非都需要重做，但应保证同一机制的输入足够广，且至少一侧可以过渡使用。参见指南 §3.3、§4.3。
5. **进攻端显著拥挤。**手里剑、空手道、奈落和大量“其他”牌都能提供伤害或伤害放大；防御和运转虽然存在，但更常由少量导出牌、能力牌或条件牌承担。未来新增卡若继续优先做攻击，会增加同维度竞争而非答案重合。
6. **奈落更像跨牌池终局状态，而不是独立卡牌体系。**卡牌侧只有 4 个条目，其中 2 张普通奖励金卡；完整入口、退出和保护还依赖遗物、药水与全局规则。把它当作稀有爆发层是成立的，把它和手里剑、茶道、空手道并列计算体系密度则会高估其卡牌侧完整度。

## 3. 机制与体系审计

### 3.1 手里剑：最完整的主体系

**底层行为。**普通手里剑是 0 费攻击，造成伤害、抽 1 张牌并消耗；强·手里剑提高伤害但保留同样的抽牌与消耗结构。它同时影响攻、牌、出牌次数和消耗堆，天然具有多层接口。

**结构评价。**

- 输入覆盖白、蓝、金多个稀有度，既有直接生成，也有放入手牌、抽牌堆、弃牌堆和回合开始生成，进入体系的路径充足。
- 输出包含按手牌数量增伤、转化为强·手里剑、自动重复打出、转化为格挡，以及向茶道、空手道导出。
- `刃舞`的“抽牌并丢弃非手里剑”使体系不只是生成 Token，还能围绕牌堆密度和过滤运作，属于自然接口。
- 体系自带抽牌和零费出牌，容易与攻击次数、临时力量、空手道等机制形成乘法增长；测试时应重点关注手牌上限、自动打出、重复生成和每次命中触发的组合。

**风险。**

- Token 同时提供伤害、过牌和消耗触发，生成数量的实际价值明显高于单纯“0 费小伤害”；不能只按伤害标准值估价。
- 多个核心导出集中在金卡，普通局中可能出现“体系自身很完整，但与职业其他主题的连接随机性较高”。
- `备刃`、`无星之夜`、`机关`等牌在输入不足时偏弱，应通过奖励模拟确认第一张组件的风险是否合理。

### 3.2 茶道：最接近独立新系统的机制

**底层行为。**茶道作为保留状态牌停留在手中；回合开始提供能量；玩家花费能量时，最先进入手中的茶道逐步提高费用，到 3 费时被弃置。它天然带来能量收益、手牌占用、顺序、弃牌时点和循环速度之间的权衡。

**结构评价。**

- 生成路径跨初始、白、蓝、金与负面效果触发，主机制有足够存在感。
- 输出覆盖手牌茶道数量、战斗内累计生成量、弃置、消耗、消耗堆数量与抽到状态牌，多角度解释同一个系统，整体性较好。
- 茶道能导向攻（`暗杀拳`、`煎茶岚`）、防（`息守`、`献茶`、`茶盾`）、牌（`哐哐喝茶`）和空手道（`殴人茶`），是当前四维覆盖最完整的体系。
- 茶道本身不是被动计数器：持有更多茶道会占手牌，兑现能量又会推动弃置，符合指南 §4.4 对“系统”的要求。

**风险。**

- 多种输出读取的不是同一个“茶道量”：手牌数量、生成总数、弃置、消耗和消耗堆彼此不同。丰富性是优点，但若卡面反馈不足，玩家可能难以预测哪些组件真正兼容。
- 多张输出裸值偏低或完全依赖茶道，第一张输出组件的抓取风险较高；应保证普通稀有度的输入密度足以支撑。
- 茶道占用手牌却又是能量来源，若通用过牌不足，可能形成“有费但缺可用牌”；若过牌和生成过强，又可能快速触发身法、手里剑或攻击次数的复合上限。

### 3.3 空手道：天然的多段与结算体系

**底层行为。**空手道是可施加给敌人或自己的层数。目标受到有威力攻击并实际损失生命时，额外失去等同当前空手道的生命，随后层数减少 1。它天然偏好多段攻击，并把“层数、命中次数、是否未格挡、何时清空”变成可操纵资源。

**结构评价。**

- 低稀有度已有稳定输入，初始牌也给玩家展示机制，不依赖先抽到金卡才认识体系。
- `掌打`、`砰砰拳`、`龙卷拳`和手里剑等多段 / 高频攻击自然充当输出，不需要每张牌显式写“若有空手道”。这是当前最强的涌现接口之一。
- `终结`把层数一次性转为无视格挡伤害，`后拱桥`把全场标记导向能量，`无中生盾`把触发导向防御，具备结算、导出和回路层次。
- 自身空手道是可被敌方命中兑现的负担，也能被清除或作为代价使用，使同一资源同时具有风险与收益。

**风险。**

- 同一标记同时存在于敌我双方，且常规触发与`终结`、`后拱桥`的清空规则不同，需要极清晰的预览和教学。
- `倍劲`、持续施加与多段攻击可能形成高倍数链条；不能只看单张牌的层数或伤害。
- 防御导出较依赖能力牌或高稀有度牌。若玩家没有拿到`无中生盾`、`道墙`等，空手道仍主要挤在进攻维度。

### 3.4 奈落：爆发层与代价层

**底层行为。**进入奈落后，每次打出攻击会生成燃烧；有威力攻击命中时对所有敌人造成额外生命损失。燃烧在回合结束时造成全场生命损失并消耗。系统把攻击次数转为群体爆发，同时把未来抽牌和生命压力作为代价。

**结构评价。**

- 奈落与手里剑、多段、零费攻击天然协同，也与状态牌、消耗和茶道呼吸形成接口。
- `赤黑`提供攻击免费化，直接服务爆发回合；`奈落回血`提供进入形态时的防护；`一心同体`把后续进入奈落的尝试转为能量、抽牌和奈落生命，具有终局回路性质。
- 机制不需要大量专属监听器即可工作，主题明确。

**风险。**

- 卡牌入口很少且集中于金 / 先古，卡牌侧不宜按完整主体系要求评估。
- 真实强度高度依赖遗物、药水、退出条件与燃烧处理，单看 4 张卡会遗漏系统上下文。
- 与高频攻击结合时，收益和燃烧生成同时爆发；应按整场战斗中的牌堆污染、退出时点和全体生命损失测试，而不是只看一回合伤害。

### 3.5 “其他”：必要的桥梁与整体性缺口

“其他”包含可靠过渡牌、跨体系桥梁、附魔、回避、负面效果、追击和牌堆复用。它们对牌池不可或缺，但也最容易成为互不相关的机制集合。

**积极部分。**

- `忍者冷漠`与`回收`提供蓄势和弃牌堆复用，能进入多种卡组答案。
- `我打`、`进化`、`血泪`、`拨牌`把茶道、状态或空手道行为导向其他维度。
- `连劲`读取“每次造成伤害”，与手里剑、多段、奈落的共同主题自然连接。
- `锁定`和`杀气`同时处理攻防时点，符合“正确释放窗口”的职业气质。

**风险。**

- 负面效果收益存在多个局部监听器：`不净之炎`、`自虐爽`、`进化`等读取不同事件，可能形成一个隐约的状态主题，却缺少统一资源流。
- `身法`要求单回合抽到大量牌后才提供回避，是典型战未来输出；它可由手里剑和茶道运转兑现，但 3 费启动会进一步加重首轮负担。
- `回避`、`追击`、`完美契合`等系统若只有少量牌参与，应明确它们是系外工具还是未来主体系，避免同时占用奖励空间却无法形成足够重合。

## 4. 四维、启动与成长

| 能力 | 当前主要来源 | 判断 |
|---|---|---|
| 进攻 | 手里剑 Token、空手道触发与结算、茶道数量输出、奈落、多段与临时力量 | **充足且拥挤。**未来新增应优先证明其不是重复攻击答案。 |
| 防御 | 手里剑转格挡、茶道条件格挡、空手道触发格挡、回避、减伤、虚弱与反弹 | **有覆盖但偏条件化。**多张高价值防御依赖体系输入或能力启动。 |
| 过牌 | 手里剑自带抽牌、刃舞过滤、茶筅斩、冷泡、哐哐喝茶、茶道三昧、身法前置需求、回收与回弹 | **体系内强、通用性中等。**进入手里剑后运转很强，体系外的大幅过牌较少。 |
| 能量 | 茶道、后拱桥、我打、忍者问候、舍力、血泪、追击、一心同体 | **数量充足但代价差异大。**与过牌能否同步兑现是测试重点。 |
| 启动 | 低费生成、预见、零费工具；同时存在大量 1～3 费能力与金卡桥梁 | **主要风险。**需要测首轮和第二轮，而非只看成型上限。 |
| 成长 | 各体系能力、累计茶道、持续空手道、身法、奈落和回路牌 | **上限充足。**部分成长集中在稀有牌，稳定成型程度需数据验证。 |

这一结构基本满足指南 §2.3 的维度覆盖要求，但“有牌”不等于“能在压力到来前兑现”。后续测试应把启动速度作为独立能力，而不是笼统算进过牌或成长。

### 5.2 茶道（21）

## 6. 资料一致性与核验记录

### 6.1 项目内差异

- `card-catalog.md` 中 `BladeDance/BladeDance` 只有标题行，没有效果；源码 `Cards/Uncommons/BladeDance.cs` 与中文本地化显示其实际名称为**刃舞**，效果为造成伤害、抽牌并弃掉抽到的非手里剑牌。
- `card-catalog.md` 中 `Footwork/Footwork` 只有标题行，没有效果；源码 `Cards/Rares/Footwork.cs` 与中文本地化显示其实际名称为**身法**，效果为当回合抽牌达到阈值后，在回合结束获得回避。
- 本文以源码与本地化补全审计信息，但按任务边界不修改卡牌目录。

### 6.2 复杂机制抽查

以下结论已直接核对项目源码：

- 茶道的回能、随能量花费升费、到 3 费后弃置：`Content/ChadoCombatRules.cs`。
- 空手道触发额外伤害、随后减少 1 层，以及触发防御导出：`Powers/KaratePower.cs`、`Content/NinjaSlayerActions.cs`。
- 奈落的攻击生成燃烧与攻击命中造成全体生命损失：`Powers/NarakuPower.cs`。
- 身法统计每回合抽牌并在回合结束给予回避：`Powers/DrawForEvasionPower.cs`。
- 茶盾、终结、手里剑大劈、暗杀拳等动态公式与卡牌目录附录一致。

### 6.3 原版机制参照

为了确认“底层机制会自然产生跨卡互动”这一评价标准，可参照本地原版代码导出中的下列实现：

- 生成攻击与攻击次数：`Slay the Spire 2/src/Core/Models/Cards/Shiv.cs`、`BladeDance.cs`。
- 抽牌、弃牌与过滤：`Acrobatics.cs`、`CalculatedGamble.cs`。
- 消耗与防御 / 过牌导出：`FeelNoPain.cs`、`DarkEmbrace.cs`。
- 格挡向进攻导出：`BodySlam.cs`。
- 抽牌或出牌次数向防御导出：`Afterimage.cs`。

这些类只用于确认互动结构与事件语义；本文没有把原版具体数值直接作为 Ninja Slayer 的平衡基准。

## 7. 原则性改进方向

### 高优先级

1. **正式确定一句职业玩法主题。**建议以“暂存资源＋高频攻击＋择机兑现”为候选，用它筛选未来卡牌：新牌必须深化该主题、连接现有系统或明确承担过渡补缺职责。
2. **降低跨体系连接对金卡的依赖。**不必下放现有金卡效果，但未来普通牌应优先提供可过渡的软连接，让手里剑、茶道、空手道在蓝白奖励中也经常同时相关。
3. **把启动速度列为独立测试指标。**重点测试能力牌较多的牌组在第一轮、第二轮的防御、过牌和能量同步程度，并与成型上限分开记录。
4. **暂停扩张新的孤立小机制。**在回避、追击、负面效果、完美契合等现有接口没有获得清晰定位前，新增卡优先深化或连接已有系统。

### 中优先级

1. **统一茶道状态语言与反馈。**明确区分“手中茶道数、生成总数、弃置、消耗、消耗堆数量”，让玩家能在奖励和战斗中准确判断协同。
2. **检查纯战未来牌的第一张抓取风险。**尤其关注依赖生成、弃置、消耗、未格挡命中或高抽牌阈值的能力；输入或输出至少一侧应能安全过渡。
3. **控制进攻端继续膨胀。**未来攻击牌优先承担桥梁、过滤、启动或防御导出，而不是再建立独立伤害标签。
4. **把奈落按“稀有爆发层”评估。**测试时纳入遗物、药水、退出规则和燃烧处理，不以 4 张卡的数量判断其完整性。

### 低优先级

1. **建立项目自己的标准值表。**按费用、稀有度、即时 / 延迟、生成牌和条件牌分别统计，并用实际遭遇校准。
2. **记录奖励决策质量。**除胜率外，观察三选一中可考虑选项数量、跳过率、首张体系组件选取率和桥梁牌出现时点。
3. **修复文档生成缺口。**后续维护 `card-catalog.md` 的生成流程时，确保自定义类名带前缀的刃舞、身法也能正确写入中文名称与效果。

## 8. 后续测试建议

本文没有玩家样本与遭遇数据，因此以下判断必须由测试验证：

- 以“首轮可用攻、防、过牌、能量”和“能力启动完成回合”记录启动速度。
- 统计每次奖励中 0、1、2、3 张可考虑牌的比例，检查整体性是否真的产生复杂选择。
- 分别记录第一张输入、第一张输出和第一张桥梁牌的裸抓率与跳过理由。
- 对手里剑＋空手道＋连劲、茶道＋身法、奈落＋手里剑等乘法链做极端测试。
- 针对群怪、成长敌人、塞牌敌人、首轮高压和长战分别验证四维覆盖。
- 调整前先确认问题属于数值、输入密度、桥梁稀有度、启动速度还是机制结构，避免只用数值修补结构缺口。
