# 卡组构筑游戏卡牌设计指南

> 来源：对“杀戮尖塔卡牌设计解析”系列视频自动字幕的整理与方法论提炼。  
> 用途：为重策略 Roguelike 卡组构筑游戏设计职业牌池、体系和单卡时提供统一语言、流程与检查标准。  
> 边界：本文不是攻略，也不把某一作品的具体数值当成跨项目通用答案。与 Ninja Slayer 当前牌池有关的评价见 [牌池审计](card-pool-audit.md)；从原著事实到角色机制的项目专用推导见 [原著卡牌设计证据](lore/design-evidence.md)。

## 2. 策略从资源分配开始

### 2.1 核心定义

**可复用原则：策略是有限资源在不同目标与能力之间的分配方案。**

卡组构筑游戏中的资源不只有能量。至少应同时观察：

- 战斗内资源：能量、手牌、抽牌次数、牌位、生命、格挡、回合数、目标与出牌顺序。
- 战斗间资源：选牌机会、金币、升级、删牌、路线、药水、遗物与可承受的生命损失。
- 信息资源：敌人意图、牌堆顺序、已知遭遇与玩家对未来风险的判断。

设计一项选择时，应先问“玩家在分配什么”，再问“这项分配产生了哪些互斥方案”。如果不存在有限资源、机会成本或不确定性，表面上再复杂的效果也未必形成策略。

### 2.2 能力维度

“能力维度”是按功能而不是按卡牌名称对玩家能力进行分类。对典型尖塔式战斗，可使用四个基础维度：

| 维度 | 目标 | 常见手段 |
|---|---|---|
| 进攻 | 在敌人成长或局势恶化前结束战斗 | 直接伤害、持续伤害、易伤、力量、多段、处决 |
| 防御 | 降低当前及长期生命损失 | 格挡、减伤、虚弱、回避、回复、控制 |
| 过牌 | 提高每回合看到正确牌的概率并加速循环 | 抽牌、检索、预见、过滤、弃牌、牌堆操纵 |
| 能量 | 提高一回合能够支付的行动量 | 回费、减费、储能、零费化、延后支付 |

进攻和防御直接影响胜负，可称为**决胜维度**；过牌和能量主要放大前两者，可合称为**运转维度**。

一张牌可以同时属于多个维度。例如“获得格挡并抽牌”同时补防御和过牌；“失去生命并获得能量”则是牺牲防御换取能量。维度标签描述的是功能，不是牌的永久身份。

### 2.3 为什么四个维度都要有压力

**可复用原则：关卡应让每个重要维度都有不可忽视的需求。**

- 持久生命损失让防御不能被长期忽略。
- 会成长的敌人、群怪和高压阶段让纯防守不能无限拖延。
- 敌人塞牌、状态与关键回合要求玩家提高过牌和稳定性。
- 手牌与能量互相制约：只有过牌会无费可用，只有能量会无牌可打。

运转可以看作攻防的乘数，但这个乘数受“过牌”和“能量”中较弱的一侧限制。因此，评价牌池时不仅要问每个维度有没有牌，还要问这些能力能否在真实战斗中同时兑现。

**作者分析 / 推论：**《杀戮尖塔》通过敌人意图交替、敌人成长和多敌人超额总伤害，使进攻能够间接转化为防御，从而避免只堆防御成为稳定最优解。

## 3. 单卡强度：标准值不是答案，而是尺子

### 3.1 建立项目自己的标准值

**可复用原则：为常见费用和效果建立内部基准，用它比较机会成本，而不是机械套公式。**

标准值应由本项目的起始牌、普通奖励牌、敌人曲线和实际测试共同校准。至少记录：

- 各费用下的单体伤害、群体伤害和格挡基准。
- 抽牌、回费、减费、弱化与控制的大致换算关系。
- 升级通常能增加多少即时价值、稳定性或协同上限。
- Common、Uncommon、Rare 的职责差异，而不只是强度差异。

视频中的具体标准值例子来自特定版本和语境，且部分数字受字幕质量影响，属于**待核实细节**。设计时应重新测量当前项目，不能直接照抄。

### 3.2 偏离标准值的四种主要理由

一张牌高于或低于基准，应能回答“差值去了哪里”。常见理由有：

1. **负面代价**：失去生命、塞入废牌、消耗其他资源、延迟生效。
2. **释放条件**：牌型、目标状态、回合窗口、牌堆位置或必须先启动。
3. **附加价值**：抽牌、保留、过滤、跨维度补强、长期成长。
4. **协同潜力**：裸用较弱，但在正确结构中能够把机制多次放大。

无色牌、初始牌、生成牌和状态牌处在不同获取环境中，不应与普通奖励牌共用完全相同的基准。初始牌可以有意偏弱以制造成长空间；稀有获取的牌可以更高效；生成牌则必须把“生成成本、手牌占用和触发次数”一起计算。

### 3.3 平衡真正保护的是什么

**可复用原则：平衡的目标不是让牌完全一样强，而是避免某个选项在绝大多数局面中无条件支配另一个选项。**

如果裸强牌可以无条件超过需要构筑的协同牌，玩家对协同、路线和未来风险的判断就会被“有没有抽到强牌”的运气压过。反过来，如果所有协同组件在未成型前都是伤口，玩家抓第一张组件的风险也主要由未来是否发牌决定。

较稳健的做法是让一组协同的输入或输出中，至少一侧在无配合时接近过渡牌水平；另一侧可以更依赖未来，但应有清晰、可达且不只依赖唯一卡牌的兑现路径。

## 4. 协同：卡组突破单卡基准的主要方式

### 4.1 输入与输出

**协同**指两个或更多组件组合后，产生大于各自独立价值之和的效果。

- **输入**：产生、累积或改变某种机制，例如生成手里剑、施加标记、弃牌、获得力量。
- **输出**：利用或响应该机制，例如按手里剑数量增伤、按标记结算、弃牌后回费、多段攻击利用力量。

输入与输出是相对于某一条协同关系的角色，不是卡牌永久属性。一张牌可以在前一段关系中作为输出，又在下一段关系中成为输入。

### 4.2 协同链、回路卡与导出卡

- **协同链**：A 产生机制，B 将其转化为第二种资源，C 再利用第二种资源。链条中的乘数可能叠加或相乘，因此上限远高于单次联动。
- **回路卡**：既被核心机制加强，又会继续加强该机制，使协同能够自我放大。回路卡很容易产生指数成长，应重点检查启动门槛、循环速度和上限。
- **导出卡**：把一个体系在某维度积累的强度转化到另一个维度，例如把格挡转为伤害、把攻击次数转为防御。导出卡能弥补短板并促进体系结合，但过强时会让整个被导向维度失去构筑必要性。

### 4.3 自然协同与人为监听

**可复用原则：优秀的核心机制应由自身性质指示至少一种协同结构。**

自然协同来自机制本身：

- 加法增益自然偏爱多段和高频触发。
- 抽牌自然需要能量或减费，也需要卸牌避免手牌溢出。
- 弃牌既是手牌过滤，也能成为抽牌与回费之间的交换接口。
- 消耗会缩小战斗牌组，天然连接循环、废牌处理和“被消耗时”收益。
- 储存在手中的资源天然产生手牌占用、保留、释放窗口与牌堆循环问题。

人为监听通常写成“每当机制 X 发生，获得任意效果 Y”。它不是必然坏设计，但如果一个体系几乎只能依靠这种句式成立，往往会出现：

- 输入和输出关系过于显眼，构筑像按说明书配对。
- 把 Y 替换为伤害、格挡或抽牌都成立，机制缺少独特性。
- 组合模式单一，较难产生意料之外的涌现协同。
- 没有监听器时输入没有用途，监听器没有输入时又是纯战未来牌。

设计机制时，先列出它在没有任何专属触发器时会怎样影响能量、手牌、牌堆、目标、频次、时间和风险；再从这些真实影响中发现协同。只有确有缺口时再加入监听式输出。

### 4.4 底层系统与新角色系统

最容易自然协同的机制来自游戏已有底层规则，例如伤害次数、牌堆移动、弃牌、消耗、费用和格挡。如果这些空间已经被充分开发，可以为职业增加一个真正的新系统：

- 它应有自己的状态、资源流或区域，而不只是一个新关键词计数器。
- 它应允许多种输入、输出、时间窗口和失败方式。
- 玩家即使没有抽到专属核心牌，也能感受到它对基础战斗的真实影响。
- 大部分职业牌应从不同角度解释这个系统，而不是只在少量牌上点到为止。

## 5. 体系结构与体系外卡

### 5.1 体系不是互斥流派

**可复用原则：体系应被视为卡组组件，而不是要求玩家从开局押注到底的互斥路线。**

一个体系通常围绕某个能力维度上的核心机制展开，包括：

1. 能独立工作的基础输入。
2. 将输入高效转化为价值的基础输出。
3. 少量回路卡，用于提高成长上限。
4. 少量导出卡，把体系优势送往其他维度。
5. 过渡或桥梁卡，使玩家能在体系未成型时进入或离开该体系。

不同体系若位于不同能力维度，能够通过互补短板自然结合；如果多个体系都只提供相同功能，它们更容易争夺牌位并稀释彼此的协同密度。将体系放在不同维度只是必要条件之一，不保证它们一定结合自然；它们的核心机制仍需存在真实接口。

### 5.2 三类体系外卡

体系外卡用于扩展答案空间，可分为三类：

1. **单卡支柱**：需要苛刻条件，但达成后可独立支撑一个维度。通常适合进攻端；若轻易支撑无限运转或绝对防御，会破坏战斗压力。
2. **系外协同卡**：不属于主体系，却能复用、放大或连接多个机制。它们最适合制造玩家自行发现的组合。
3. **过渡 / 补缺卡**：协同很少，但裸用可靠，用于度过前期、针对敌人或补齐当前短板。

体系外卡不是“没有定位的剩余牌”。每张都应明确回答：它增加了哪种卡组答案、改善了哪个决策，或为哪个阶段提供安全垫。

## 6. 牌池结构：让玩家持续拥有选择

### 6.1 策略与随机性的五个控制点

**作者分析 / 推论：**重策略卡组构筑游戏需要同时控制五个因素：

1. **卡牌平衡**：减少抽到无条件强牌造成的胜负波动。
2. **体系数量**：同一牌池的独立体系越多，抽中特定体系的概率越低。
3. **拿牌机会**：密度应匹配难度上升速度，总量应允许优秀玩家在坏运气下仍有构筑路径。
4. **初始限制**：起始套装不应过早锁死唯一体系，除非这是特定模式的明确目标。
5. **卡池结构**：不同可通关答案之间应有足够重合，使玩家在中后期仍能转向。

这些因素是联动的。减少体系数量却同时减少拿牌机会，不一定降低运气；增加大量选牌但多数选项与当前卡组无关，也不会自动增加策略。

### 6.2 卡组答案与重合度

把一种能处理目标环境的成型结构称为一个**卡组答案**。一个健康牌池应同时提高：

- **答案数量**：有多种方式满足攻、防、运转和成长需求。
- **答案重合度**：同一张牌或同一组件能出现在多种答案中，让早期选择不会立即锁死后续路线。

提高重合度的常用手段包括：

- 让体系使用基础机制，从而能与体系外卡自然互动。
- 用导出卡和桥梁卡连接不同能力维度。
- 设计可服务多个体系的复用、过滤、频次和牌堆操纵工具。
- 避免每个体系都依赖唯一核心卡才能工作。
- 让同一功能存在不同代价和节奏，而不是简单的上下位替代。

### 6.3 卡池整体性

**作者分析 / 推论：一个优秀牌池是对同一主题的多种诠释。**

这里的“主题”不是美术题材，而是职业持续面对的核心玩法问题，例如：

- 如何协调抽牌、回费与卸牌。
- 如何在首轮启动与长期成长之间分配牌位。
- 如何管理一种新增资源、区域或角色系统。

当大部分牌都在回答同一个问题时，三选一中更容易同时出现多张“看起来都有用”的牌，玩家必须比较当前风险、未来上限和机会成本。相反，如果牌池只是若干互不相关的小体系并排放置，奖励更容易变成“一张属于当前体系、两张无关”，认牌之后决策会迅速变浅。

卡池整体性有一种例外路径：多个体系虽然没有单一主主题，但它们都来自游戏底层逻辑，并且接口足够自然、结合足够紧密。这时整个牌池的共同主题就是“基础系统本身”。

## 7. 新卡与新体系设计流程

### 第一步：写清设计目标

- 目标玩家是谁，预期难度和可重玩性是什么？
- 这张牌解决哪个战斗阶段、敌人压力或牌池缺口？
- 它要增加一个新答案，还是提高已有答案之间的重合度？

### 第二步：确定牌池主题与卡牌职责

用一句问题描述职业主题，避免只写背景设定。例如：“玩家如何把暂存于手中的资源，在正确回合转化为攻防与运转？”

给卡牌指定职责：输入、输出、回路、导出、桥梁、单卡支柱、系外协同或过渡补缺。若无法指定，先不要设计具体效果。

### 第三步：选择核心机制

- 优先从已有战斗系统中寻找尚未充分开发的关系。
- 若创建新系统，先定义资源流、状态变化、时间窗口、失败方式和与基础规则的接口。
- 写出至少三种不依赖“每当 X 就获得 Y”的自然协同。

### 第四步：映射能力维度

标记卡牌直接覆盖的维度、间接导出的维度，以及它牺牲的维度。确认体系不是只在一个维度内部自我放大，而完全依赖其他体系兜底。

### 第五步：确定输入、输出与兑现路径

- 输入是否有基础用途？
- 输出在没有专属输入时是否完全失效？
- 玩家从第一张组件走到成型需要多少张特定牌？
- 是否存在多个输入或输出，避免唯一核心卡依赖？
- 回路或协同链的上限由什么约束？

### 第六步：用标准值估价

分别估算裸用、常见配合和极端上限。把生成成本、手牌占用、启动回合、目标限制、升级和遗物放大一起计算。标准值只能指出异常，最终数值必须通过遭遇测试和玩家数据调整。

### 第七步：放回整个牌池

模拟三选一而不是只看理想卡组：

- 未进入该体系的玩家为何会考虑这张牌？
- 已进入体系的玩家是否无脑必拿？
- 它与哪些现有答案重合，又会排斥哪些答案？
- 同稀有度、同费用、同职责是否已有更稳定的替代品？

### 第八步：测试并迭代

至少记录选取率、跳过率、升级率、出现时点、胜率相关性、无配合裸抓率和关键敌人表现。数据用于发现问题，不能单独证明因果；调整时要结合录像、牌组状态和玩家决策理由。

## 9. 四级检查清单

### 9.1 单卡

- [ ] 裸用时有可解释的价值，或其战未来风险与收益相称。
- [ ] 费用、牌位、启动时间和附加效果都已计入标准值比较。
- [ ] 高于基准有真实代价或条件，低于基准有明确附加价值或协同。
- [ ] 升级改变价值、稳定性或路径，而不是无目的地堆数字。
- [ ] 不会在绝大多数局面中成为同职责牌的无条件上位替代。
- [ ] 极端复用、多段、减费和循环情形已被检查。

### 9.2 机制

- [ ] 机制对基础战斗有实际影响，而不只是供触发器读取的标签。
- [ ] 机制自身能指示至少一种自然协同结构。
- [ ] 有多个输入和输出，不依赖唯一核心卡。
- [ ] 输入和输出至少一侧在未成型时仍有用途。
- [ ] 失败方式、启动门槛和成长上限清晰。
- [ ] 监听式协同是补充，不是体系成立的唯一理由。

### 9.3 体系

- [ ] 核心机制和主要能力维度明确。
- [ ] 包含基础输入、基础输出、过渡入口和合理的成长手段。
- [ ] 回路卡不会过早形成无约束指数成长。
- [ ] 至少有一个通往其他能力维度的自然接口或适度导出。
- [ ] 能和其他体系互补，而不是仅争夺相同功能和牌位。
- [ ] 不要求玩家从开局开始只抓同标签牌。

### 9.4 整个牌池

- [ ] 能用一句玩法问题说明共同主题。
- [ ] 大部分牌是对主题的不同诠释，而不是互不相关的小包。
- [ ] 进攻、防御、过牌、能量以及启动、成长均有可达方案。
- [ ] 体系数量与拿牌机会相匹配。
- [ ] 起始套装不会无意中锁死唯一发展方向。
- [ ] 多种卡组答案之间有足够重合度。
- [ ] 三选一经常产生多个可考虑选项，而不是明显的一选二废。
- [ ] 导出卡补短板但不让整个维度失去构筑必要性。
- [ ] 过渡卡、系外协同卡和单卡支柱各自有明确职责。
- [ ] 平衡结论来自遭遇测试和数据，而不是只看理想连招。

---

## 10. 使用原则

这套框架服务于“重策略、可重复游玩”的设计目标，不是评价所有卡牌游戏的唯一标准。叙事、角色扮演、短期爽感或高随机性可以是更重要的目标；此时可以主动放宽整体性、初始限制或平衡要求，但应把它视为明确取舍，而不是无意产生的结果。
